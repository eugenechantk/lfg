### 2026-07-01 — FlowDeck sim guard thrashed after `cd` in Bash commands

**What happened:** Between flowdeck calls I ran `cd .../ios/LFG && grep ...` and `cd .../lfg && bun ...`. The sim guard keys off cwd; each shift made it create a NEW session sim (ios-3dbf7e5c → LFG-3dbf7e5c), so `-S <old udid>` got blocked and I had to migrate the whole run/install onto the newly-demanded sim.
**Why this was wrong:** Memory `flowdeck-sim-guard-cwd` already says "don't cd between flowdeck commands." I used absolute paths for flowdeck but still `cd`'d in *other* commands in the same shell, moving the guard's cwd.
**What better looks like:** Never `cd` in ANY Bash command during a flowdeck session — always use absolute paths (Read/Grep tools instead of `cd && grep`). The shell cwd is shared state the guard reads.

### 2026-07-01 — Misread elapsed time between tool calls

**What happened:** Turned on the mock outage, then in a later Bash call did `sleep 4; screenshot` and saw "Offline" — briefly suspected the debounce fix wasn't working. Actually ~30s of real wall-clock passed between the two tool calls (my reasoning time), during which the 3s poll failed ≥3× and correctly flipped to Offline.
**Why this was wrong:** I reasoned about elapsed time as if only the in-command `sleep` counted, ignoring that the app keeps polling during the gaps between my tool calls.
**What better looks like:** For time-sensitive live verification, put the trigger + wait + capture in ONE Bash command so elapsed time is controlled, rather than spread across calls with unbounded reasoning gaps between them.
