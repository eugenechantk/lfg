## Log

### 2026-07-02 — FlowDeck build flag confusion

**What happened:** First `flowdeck build` call used `--destination "generic/platform=iOS Simulator"` (xcodebuild vocabulary). FlowDeck rejected it and printed help; second try `--simulator "iPhone 16 Pro"` failed (not installed); third `--simulator "iPhone 17"` worked.
**Why this was slow:** Reached for raw-xcodebuild muscle memory instead of FlowDeck's own flag surface. Cost two throwaway invocations.
**What better looks like:** For FlowDeck builds, use `--simulator <name|udid>` (or `--device`), and run `flowdeck simulator list` first to pick an installed device type. Never pass an xcodebuild `--destination` string.

### 2026-07-02 — Verified the primitive, not the stale endpoint

**What happened:** The fork endpoint was added but the running `lfg serve` predates it, so I couldn't curl the endpoint without a restart (which drops in-memory session tracking others depend on). Instead I isolated the ONE genuinely new risk — the `claude --resume … --fork-session` primitive — and exercised it directly in a throwaway tmux pane, confirming a new id is minted and the source is untouched, resolvable via the same pidfile path the endpoint uses.
**Why this was right:** The endpoint wraps the production-tested resume path; the only new behavior is the flag. Testing the flag against a real transcript is the real seam without paying the restart cost. Matches [[verify-real-seam-not-mocks]].
**What better looks like:** When an endpoint is added but the prod server is stale, don't restart to test — isolate and exercise the smallest new primitive against real data, and flag the restart as a separate deploy step.
