### 2026-07-01 — codex file-open 404 root cause was temp-dir roots, not stale build

**What happened:** "Files 404 for codex sessions." A June 28 diagnosis had attributed a similar 404 to a stale on-device build. This time it was scoped to codex only (Claude worked on the same build), so it had to be a code difference. Root cause: codex `--yolo` saves screenshots to `/tmp/*.png` and `$TMPDIR/.../T/*.jpg` and presents those paths; `/api/file` only served `$TMPDIR/lfg-uploads`, so cards 404'd. Fixed by adding `/tmp`, `/private/tmp`, `tmpdir()` to the served roots (homedir was already served, so no new exposure).
**Why it matters:** The "scoped to codex" detail was the key that ruled out the prior stale-build explanation. Getting the literal failing paths from transcripts + live-endpoint tests (per [[ground-truth-before-hypothesizing]]) beat re-theorizing.
**What better looks like:** When a "previously fixed" bug recurs with a new scope qualifier, treat the qualifier as a discriminator — find what differs between the working and broken cohort before reusing the old diagnosis.
