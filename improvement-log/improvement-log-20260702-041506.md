## Log

### 2026-07-02 — TestFlight deploy grabbed system Ruby 2.6 instead of rbenv 3.2.7

**What happened:** `bundle exec fastlane ios deploy_testflight` failed immediately with `Could not find 'bundler' (2.4.19)` because the non-interactive Bash tool shell resolves `bundle`/`ruby` to `/usr/bin/*` (system Ruby 2.6), not the rbenv global 3.2.7 the Gemfile.lock needs. The earlier deploy today worked because it ran in a shell that had rbenv on PATH.
**Why this was slow:** Spent a couple of calls locating the right Ruby (checked homebrew rubies, rbenv versions dir) before realizing the rbenv shim was already 3.2.7 and the only issue was PATH ordering.
**What better looks like:** For any `bundle`/`fastlane`/ruby-tool invocation in this repo, prepend `export PATH="$HOME/.rbenv/shims:$PATH"` in the same command (shell env doesn't persist across Bash calls). Now captured in memory `lfg-testflight-ruby-path` and MEMORY.md so the next deploy skips the rediscovery.
