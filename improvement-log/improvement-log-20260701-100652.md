### 2026-07-01 — Sim-guard cwd basename mismatch cost a rebuild/install
**What happened:** My first `flowdeck run` used `cd /Users/.../lfg/ios; flowdeck run` so the per-session sim guard computed the sim name from basename `ios` → `ios-118dc4a7` (23937414) and installed the app there. Subsequent `flowdeck ui simulator …` commands ran from the persistent cwd (project root, basename `lfg`) → guard demanded `lfg-118dc4a7` (9D7C189B). I had to reinstall/relaunch the app on 9D7C189B to do UI work on the same sim the guard would allow.
**Why this was wrong:** Memory `flowdeck-sim-guard-cwd` says the guard keys off cwd — but I only had "don't cd between commands." The sharper rule: the guard's sim NAME derives from the **cwd basename**, so running some flowdeck commands from a subdir (`ios/`) and others from the repo root splits you across two session sims. For a repo where the Xcode project is in a subfolder, pick ONE cwd for all flowdeck commands.
**What better looks like:** Run every flowdeck command (build/run/ui) from the SAME directory — ideally the repo root — and never `cd` into `ios/` for `flowdeck run`. flowdeck resolves the project via its absolute config path, so cwd doesn't need to be the project dir. Consider updating memory `flowdeck-sim-guard-cwd` to note the basename→sim-name derivation explicitly.
